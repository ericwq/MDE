## 模块命名基本约定

#### ▶[上一节](1.md)

在 Java 和 C# 两种语言中，模块的名称均采用层级结构形式 <sup>[1](#1)</sup> 。层级中的每个级别通过句点（`.`）分隔。这种名称层级通常以开发该模块的组织名称开头，并结合该组织的互联网域名构成。当使用互联网域名时，一般先写顶级域名，再写组织域名，具体如下：

```
com.saasovation // Java
SaaSOvation // C#
```

使用唯一的顶级名称，能够避免与你项目中使用的第三方模块发生命名空间冲突，也能避免你的模块被他人使用时产生此类冲突。如果你对这些最基础的约定有疑问，可以查阅相关标准 <sup>[2](#2)</sup> 。

你的组织很可能已经确定了一套顶级模块命名约定，遵循该约定保持一致性是最佳做法。
