## 第 9 章：模块 (Modules)

#### ▶[上一节](../ch8/8.md)

*<p align="right">一切胜利的秘诀，都在于对非显而易见事物的组织。</br>——Marcus Aurelius</p>*

如果你正在使用 Java 或 C#，那么你早已熟悉`Modules` (模块)，只是对它们有着另一种称呼。Java 中将其称为包 (packages)，C# 中则称之为命名空间 (namespaces)。事实上，在 Ruby 语言里，你可以使用 module 这一语言构造来为类实现命名空间的效果。在 Ruby 的场景中，DDD 的模式名称与语言构造的名称恰好一致。为了贴合我们的 DDD 语境，在大多数情况下我都会继续称其为模块。你可以很轻松地将这个名称对应到你日常使用的编程语言术语上。我不会花费太多时间从技术层面解释模块的作用，因为你很可能早已明白。
